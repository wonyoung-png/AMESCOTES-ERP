# 누락 알림 시스템 설계

## 개요
ERP 데이터를 주기적으로 스캔하여 누락/오류 항목을 자동으로 감지하고 담당자에게 알림.

## 알림 유형
| 유형 | 감지 조건 | 우선순위 |
|---|---|---|
| BOM 누락 | 생산 품목 중 BOM 없는 경우 | 높음 |
| 입고 미처리 | 예정 입고일 경과, 미처리 발주 | 높음 |
| 납기 임박 | 납기일 7일/3일/1일 전 미출고 | 높음 |
| 안전재고 미달 | 재고가 안전재고 이하로 감소 | 중간 |
| 발주 승인 대기 | 승인 대기 3일 이상 경과 | 중간 |
| 단가 미입력 | BOM 원자재 단가 없음 | 낮음 |

## 알림 채널
- **텔레그램** (대표님): 긴급/중요 알림
- **슬랙** (담당자): 일반 업무 알림

## 스케줄
| 체크 항목 | 주기 | 시간 |
|---|---|---|
| BOM 누락 | 매일 | 09:00 |
| 입고 미처리 | 매일 | 09:00 |
| 납기 임박 | 매일 | 08:00 |
| 안전재고 미달 | 매일 | 09:00 |

## 알림 메시지 형식
```
⚠️ [AMESCOTES ERP 알림]
유형: BOM 누락
품목: ABC-001 (레더 핸들)
등록일: 2026-03-20
담당자: 홍길동
→ 즉시 BOM 등록이 필요합니다.
```
