## B. OEM 6단계 파이프라인 ✅
```
①거래처 ▶ ②샘플 ▶ ③BOM·원가 ▶ ④생산발주 ▶ ⑤입고·납품 ▶ ⑥정산
```
| 단계 | 핵심 |
|------|------|
| ①거래처 | 바이어/공장/자재처 등록, 코드·중복검사, 은행정보 |
| ②샘플 | 의뢰일·단계·담당자, 파일첨부 |
| ③BOM·원가 | 자재명세 + 사전/사후원가, **제품 패킹사이즈**, OCR 이미지입력 |
| ④생산발주 | **자재조달 A(사급)/B(완사입) 분기**, 공장단가, 리오더 자동차수 |
| ⑤입고·납품 | 입고처리, **물류비 CBM 배분**(운송사 정산서 기준, 부가세 제외) |
| ⑥정산 | **미수금 관리**⭐ + **불량 차감 이월**⭐🆕 + 세금계산서 발행 |

**불량 차감 이월 로직 (신규):** 결제 후 발견된 불량 → 공장별 "차감 대기" 누적 → 다음 결제 시 자동 상계.

## C. 3-코드 축 (모든 연동의 뼈대) ✅
```
거래처코드 (NW/OS/LLL)  +  품목코드 (AT26HB001)  +  자재코드 (M01/Z01)
```
- **연결은 코드로만**, 이름은 표시용, UUID는 내부 숨김
- 품목코드 앞자리 = 워크스페이스 소속 (AT=OEM, LM=LUMEN, AE=AETALOOP)
- **현 문제**: 거래처코드 필드 2개 중복(code/vendorCode), 물류·구매가 이름으로 연결됨 → 코드 표준화 필요
- **리오더**: 품목코드로 이전 발주 검색 → `품목코드-R차수` 자동 (이미 잘 작동)
- **자재 통합발주**: 자재를 이름 아닌 **코드로 합산**하도록 수정 필요 + "미발주 소요 현황" 뷰 신설

## D. 워크스페이스 — 공유 백본 + 조립식 ✅
```
        [공유 백본 = 하나의 DB]
  거래처·품목·샘플·BOM/원가·발주·입출고·자재

   [OEM]      [LUMEN]     [AETALOOP]     [전체]
```

## G. 브랜드 사업 흐름 ✅
```
①기획(운영캘린더) ▶ ②생산(OEM 백본 그대로) ▶ ③3PL 입고 ▶ ④채널 분배 ▶ ⑤판매(자동수집) ▶ ⑥정산
```
- **생산**: LUMEN/AETALOOP 신제품도 **OEM 파이프라인 그대로** (샘플→BOM/원가→발주→입고). 먼저 그대로 쓰고 추가기능은 OEM과 엮어가며.
- **입고**: 공장 → **3PL 창고**로 입고
- **채널**: 자사몰(Cafe24) · 해외자사몰(Shopify) · W컨셉 · 29CM · 한남쇼룸 · 센텀백화점
- **배분**: 채널별 재고 분배는 **선택적**(할 때도 안 할 때도) — 단 **품절 후 추가판매 차단(재고 상한)은 필수** ⭐
- **판매수집**: ERP가 **전 채널 판매 자동 수집** (수기 X)
- **Ecount**: **완전 대체** (입출고·재고·구매·판매 전부 새 ERP로) 🔴 큰 결정

## H. 채널 관리 ✅
| 채널 | 판매데이터 수집 방식 |
|------|----------------|
| 자사몰 Cafe24 | **API 자동** (세션 연결됨) |
| 해외 Shopify | **API 자동** (세션 연결됨) |
| W컨셉 / 29CM | **매일 엑셀 다운로드 + 월정산 파일** → AI 파싱 |
| 센텀백화점 | 관리자 화면 / 월마감 파일 |
- **현 시스템**: 이지어드민(재고/출고/채널) + Ecount(회계/구매/판매) → **Ecount만 자체ERP로 대체, 이지어드민은 유지+연동** (아래 N 참조)
- ⭐ **필수 수정 — 실판매가 매출**: 지금 이지어드민이 W컨셉 **할인판매를 정상가로 잡는 에러**. 새 ERP는 **채널별 실제 받은 금액(할인반영)으로 매출 집계** 필수.

## ⭐ 브랜드 리오더 관리 (실제 시트 분석) — OEM↔브랜드 연결의 핵심
MD가 관리하는 리오더 시트(5탭: 가방/슈즈/별주/raw) 분석 결과:
- **품목코드 체계 이미 존재**: `LLL6F92SB` = LUMEN(LLL) + 26(6) + FW(F) + 품번(92) + 컬러(SB). → 이게 **품목코드 축**. raw data 탭 = 코드·원가·판매가 마스터.
- **리오더 = 배치 단위** (오더날짜별로 여러 품목 묶음), 재고분류 = **국내/중국** 생산지
- **🔴 핵심: 한 리오더에 [브랜드 재고 + OEM 중국납품]이 합산됨**
  - 예: "봉발롱(창성9차) 시어링블랙 14개 — 중국 납품 10개까지 포함한 수량"
  - → 브랜드 자체재고 + OEM 바이어 납품분을 **한 공장발주로 합쳐서 MOQ·효율 확보**
  - → **생산발주 = 다목적**(브랜드용 수량 + OEM용 수량 분리 표기) 필수
- **슈즈 = SKU(사이즈)별 관리** + 부분입고 이력 (4/7 2입고, 4/21 17입고)
- **별주(특별주문)** 유형 별도 + 공장차수(창성3차/9차) + 확인 담당자
- **생산지 = 공장별 분리**: LUMEN/AETALOOP 대부분 중국, 가끔 국내 → **발주 배치는 공장(생산지)별로 그룹**

### R1. AI 리오더 추천 = 수요예측 모델 🆕⭐
단순 평균 X. 아래 신호를 조합한 예측 필요:
- 시즌별 · **날씨(장마철 등)** · 월평균 판매 · **기획전 예정**(캘린더 연동) · 스파이크 감지
- "평소 1개 나가던 게 오늘 갑자기 20개" → 일시 스파이크 vs 실제 추세변화 구분 로직
- **가능한 이유**: 판매이력(채널) + 기획전(운영캘린더) + 재고가 **한 시스템에 통합**되니까 → 통합 ERP의 최대 payoff
- 출력: "품목 X, 재고 N일치 남음, 기획전 감안 → **OO개 리오더 추천**"

### R3. 리오더 6단계 승인 워크플로우 🆕
```
①MD 작성 → ②생산팀 납기확인 → ③MD팀 재확인 → ④(MD)팀장 승인 → ⑤디자인팀장 승인 → ⑥대표 승인
```
- **크로스팀·크로스워크스페이스 승인 엔진** 필요 (단계별 담당자·승인/반려·진행상태)
- 최종 승인 시 → 생산발주(OEM 파이프라인)로 자동 전환

## J. 브랜드 정산 ✅
- **수수료 = 고정 아님**: 할인·기획전마다 1~2% 차등 → **정산서에서 실수수료를 읽음** (하드코딩 X)
- **정산 주기 = 매월 10일 입금** → 미수금 D-day 추적 기준
- **월마감 파일 = 진실의 원천** ⭐: 각 채널 월마감 파일에 **실판매가·실수수료·실입금액 전부 있음**
- **설계**: 
  ```
  두 개의 데이터 흐름
  ① 일간: 판매·재고 모니터링 (Cafe24/Shopify API + 이지어드민 대체 엑셀)
  ② 월간: 정산·매출 확정 (채널 월마감 파일 → AI 파싱) ← 실판매가 에러 해결
  ```
- 월마감 파일 AI 파싱 → 실판매가 매출 + 수수료 차감 + 실입금(10일) 미수금 자동 집계

## K. 대시보드 — 위젯 조립형 ✅
- **고정 대시보드 X → 위젯 라이브러리에서 조립** (팀별·담당자별)
- **탭 상세 분할** + 각 화면 위젯으로 구성
- **저장 2단**: 팀 기본 대시보드(팀장 세팅) + 개인 커스터마이즈
- **권한·워크스페이스 자동 필터**: 위젯이 보여주는 데이터는 그 사람 권한/브랜드에 맞춰 필터됨
- **위젯 = 우리가 설계한 기능 전부가 위젯이 됨**:
  | 카테고리 | 위젯 예시 |
  |---------|----------|
  | 매출·정산 | 오늘/브랜드별 매출 · 채널별 수익성 · **미수금 D-day** · 실판매가 매출 |
  | 재고 | 재고 부족 알림 · 채널별 재고 · 재고자산 총액 |
  | 리오더 | **AI 리오더 추천** · 승인 대기 · 리오더 진행 |
  | 생산(OEM) | 발주 현황 · 납기 D-day · 입고 예정 · **불량 차감 대기** · 물류비 배분 대기 |
  | 운영 | 기획전 진행률 · 팀 업무 완료율 |
  | 개인 | 내 할 일 · 내 승인 대기 |
- **팀별 기본 예시**: MD=판매·재고·리오더 / 생산=발주·납기·입고 / 경영=매출·이익·미수금·재고자산

## L. 권한 — 설정형 RBAC ✅
**핵심: 하드코딩 X → 관리자(대표/팀장)가 화면에서 설정.**
- **3단계 권한 구조**:
  1. **워크스페이스 접근**: OEM / LUMEN / AETALOOP — 누가 어디 들어가나
  2. **탭 접근**: 탭별로 허용/차단 (전체 탭 설정에서 조정)
  3. **뷰어 / 에디터**: 탭마다 조회만 vs 수정 가능 다르게
- **기본값(설정으로 변경 가능)**:
  - OEM 관련(발주·원가·바이어) → **OEM 팀원만**
  - LUMEN/AETALOOP → **브랜드팀** (브랜드별 세분 가능, MD팀 등 개별 설정)
  - 원가·이익 → 해당 워크스페이스 팀 내부만 (외부팀 차단)
- **위젯·데이터 자동 필터**: K 대시보드 위젯이 이 권한을 그대로 따름
- **권한 설정 화면**: 대표/팀장이 팀별·담당자별·탭별로 뷰어/에디터 지정

## M. AI 자동화 + 알림 + 소통 ✅
**AI 기능 (대화 중 축적된 것):**
1. OCR 이미지 입력 (작업지시서 → BOM) 2. 물류비 정산서 파싱(CBM 배분) 3. 채널 월마감 파싱(실판매가)
4. AI 수요예측 리오더 추천 5. 기획전 팀별 준비 AI 제안 6. AI 어시스턴트(자연어 조회·등록)

**알림 = 개인 설정형** (위젯·권한과 동일 철학):
- 업무 탭 만들고 → 팀별 중요도 설정 → **AI가 "나한테 중요한 것" 알림 추천**
- 담당자별로 다름 · **슬랙으로 전달**
- 후보: 재고 품절임박 · 미수금 연체 · 불량 차감 대기 · 납기 D-7 · 승인 정체 · 채널 재고 불일치 · 판매 스파이크

**소통 = 맥락 코멘트 + 슬랙 연동** (별도 메신저 X):
- 각 레코드(발주·리오더·기획전)에 **코멘트 스레드 + @멘션** → 슬랙 알림
- 실시간 대화는 슬랙, 맥락·승인·기록은 ERP에 축적
- 리오더 6단계 승인 = 각 단계 코멘트로 남음 → 슬랙 알림
