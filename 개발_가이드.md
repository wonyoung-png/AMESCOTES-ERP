## 1. 무엇을 만드나

**제조(OEM) ERP + 브랜드 운영 ERP를 합친 "하나의 통합 ERP".**

| 사업 | 내용 |
|------|------|
| **OEM (AMESCOTES)** | 타 브랜드 핸드백 제조·납품 (B2B). 생산부·개발부. |
| **브랜드 (LUMEN / AETALOOP)** | 자체 브랜드 기획·판매 (B2C + 도매). MD·마케팅·비주얼·디자인·쇼룸·물류CS. |

→ 3개 **워크스페이스**(OEM / LUMEN / AETALOOP)가 **하나의 공유 데이터**를 필터해서 보는 구조.
→ 원가·샘플·BOM·발주·입출고는 **공유 백본**(제조)이고, 브랜드는 그 위에 운영·판매·정산을 얹음.

## 3. 기존 코드 = 이 저장소 (개조 대상)

**밑바닥부터 재작성 X. 이 코드를 개조해 새 통합 ERP로 진화시킴.**

- 스택: **React + TypeScript + Vite + Express + Supabase**
- 접속(개발): `npm install --legacy-peer-deps` → `npm run dev`
- 로그인: `wonyoung@atlm.kr` / `atlm2026`

### 핵심 파일
```
client/src/
  pages/           ← 화면 22개 (아래 참고)
  lib/
    store.ts         ⭐ 데이터 모델·CRUD·Supabase 연동 (제일 중요)
    supabaseQueries.ts / syncFromSupabase.ts  DB 연동
    supabase.ts / auth.ts   접속·인증
    bomExcelParser.ts       원가표 엑셀 파싱
agents/            ← AI 에이전트 (agent-team, erp-mcp-server) — OCR·조회·등록
server/index.ts    ← Express (정적 서빙 + /api/yardage/ocr)
supabase/schema.sql ← DB 스키마
```

## 4. 개조 순서 (진화 로드맵)

| 단계 | 내용 |
|:---:|------|
| **0** | 클라우드 배포 (localhost 탈출, 항상 켜짐) |
| **1** | UI 셸 개조 — BGROW 프리미엄 스킨 + 그룹 사이드바 + 워크스페이스 전환 + Inbox (기존 페이지 그대로 담기) |
