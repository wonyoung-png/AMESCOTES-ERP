# ATLM 통합 ERP — 브랜드 운영 설계서

> **상위 문서:** `DESIGN.md` (생산)  
> **범위:** 3PL 입고 ~ 채널 판매 ~ 정산 ~ MD 운영  
> **상태:** Design Lock (운영 우선순위 §0)  
> **⭐ ERP 핵심 가치 = 브랜드 운영** (생산은 백본, 운영이 North Star)

---

## 0. 전략 우선순위 (확정)

> **「브랜드 운영에 있어서 ERP 기능이 더 중요하다」** — 2026-07-10

| 구분 | 역할 | 개발 투자 |
|------|------|----------|
| **브랜드 운영** | ERP가 **존재하는 이유** — MD·채널·기획전·정산·수익 분석 | **최우선 (60~70%)** |
| **생산 (OEM 백본)** | 공급을 위한 **필수 백본** — 기존 코드 재사용·안정화 | **필수 최소 (30~40%)** |
| **OEM B2B 단독** | 유지·개선하되 **과투자 금지** | 기존 수준 유지 |

### ERP가 풀어야 할 브랜드 운영 Pain (우선순위)

| 순위 | Pain | ERP 기능 |
|:---:|------|----------|
| 1 | 채널별 **실판매가**·정산 (이지어드민 오류) | 월마감 AI 파싱 · 채널 정산 |
| 2 | MD **기획전·캠페인 손익** (BGROW campaign-pl) | 기획전 P&L · GMROI |
| 3 | **판매·재고** 한눈에 (이지어드민 분산) | 채널 판매 · 재고 조회 연동 |
| 4 | **리오더** 감(수기 시트) | AI 추천 · 브랜드 발주 연동 |
| 5 | 홀세일 **서류·정산** (해외) | 인보이스 · 패킹리스트 · 미수금 |
| 6 | 운영캘린더·팀 협업·**업무 OS** | 캘린더 · 업무허브 · 위젯 · Inbox |

**ERP 밖 (외부 툴):** CS=채널톡 · 쇼룸 POS=이지어드민

### 생산은 「최소 통과선」

브랜드 운영이 돌아가려면 아래만 **안정**이면 됨:

- 브랜드 발주 → 승인 → 생산발주 (LUMEN/AETALOOP 각각 분리)
- 3PL 입고 ↔ 발주번호
- BOM 원가 → 생산비용 집계

OEM 엑셀 작업지시서 완벽 이식·BOM UI 대수술 등은 **운영 이후**.

```
[DESIGN.md — 생산]                    [본 문서 — 브랜드 운영]
샘플·BOM·발주·승인·공장·3PL입고  →  채널·재고·판매·정산·기획·CS
         │                                    │
         └──────── 3PL 입고 확정 ──────────────┘
                        │
              이지어드민 (재고·출고 실무)
                        │
              ERP (매출·정산·분석·기획)
```

| 시스템 | 역할 |
|--------|------|
| **ERP** | 판매 집계 · 실판매가 매출 · 채널 정산 · 리오더 AI · 운영캘린더 · MD 대시보드 |
| **이지어드민** | 3PL 바코드 재고 · B2C/쇼룸/홀세일 **출고지시** · 채널 할당(오버셀 방지) |

## 2. 브랜드 운영 6단계 (기획서 G)

```
① 기획 (운영캘린더·기획전)
    ↓
② 생산 (DESIGN.md — 공유 백본)
    ↓
③ 3PL 입고 (ERP 기록 ↔ 이지어드민 바코드)
    ↓
④ 채널 분배 (이지어드민 주력, ERP는 모니터링)
    ↓
⑤ 판매 수집 (API + 엑셀 AI 파싱)
    ↓
⑥ 정산 (월마감 파일 = 진실의 원천)
```

## 2.1 MD 팀 일상 업무 (인터뷰 확정 2026-07-10)

### 조직

| 항목 | 내용 |
|------|------|
| LUMEN / AETALOOP | **같은 사람이 겸임 가능** |
| 업무 | **브랜드별 완전 분리** — 워크스페이스·발주서·데이터 혼합 금지 |
| 팀 구성 | 일부 인원은 **브랜드별 다른 팀** |

### MD 반복 업무 리듬

| 주기 | 업무 | 현재 | ERP 목표 |
|------|------|------|----------|
| **매일** | 재고 체크 | **수기** (이지어드민 등) | 재고 스냅샷 + 품절임박 알림 |
| **매일** | 리오더 시트 검토 | MD팀, 구글시트 | 리오더 탭 (카테고리 필터) |
| **주 1~2회** | 묶음 발주서 작성 | LUMEN/AE 각각 | `brand_order_batches` |
| **상시** | 기획전 대비 발주 | 캘린더 보고 **미리** 예측 발주 | 캘린더 ↔ 발주 연동 |

### 리오더 시트 (실제 탭 구조)

| 탭 (구글시트) | ERP 카테고리 필터 |
|--------------|------------------|
| **오더 집계** | `bag` (가방) |
| **슈즈 오더 집계** | `shoes` |
| **의류 오더 집계** | `apparel` |
| **raw data** | 품목마스터 동기화 (편집은 ItemMaster) |

- **갱신:** MD팀, **매일** 확인
- **발주 주기:** **주별 또는 2주 간격**
- **핵심:** 탭마다 UI만 다르고 **발주 로직 동일** — **`상품코드` 1개 = 발주 1줄**

### 발주 결정 기준 (3요소 — AI 목표)

```
판매속도  +  기획전(캘린더) 예정  +  재고
        ↓
   이번 주/2주 발주 수량
```

- 기획전마다 **할인·푸시 제품이 다름** → 캘린더에 적힌 기획전을 보고 **사전 발주**
- ERP **AI 리오더:** 위 3신호 조합 → 추천 수량 → MD가 수정 후 발주서 작성

### MD가 매일 보는 화면 (현재)

| 채널 | 도구 |
|------|------|
| W컨셉 / 29CM / 자사몰 | 각 **관리자 페이지** (분산) |
| 재고 | 수기 체크 |
| 리오더 | 구글시트 (오더집계/슈즈/의류) |

→ ERP **MD 대시보드:** 채널·재고·판매속도·기획전 D-day **한 화면** (관리자페이지 대체·집계)

### 생산팀 전달 전 MD 체크리스트

| # | 확인 항목 |
|---|----------|
| 1 | **생산 납기** (@멘션으로 생산팀 입력 → MD 참고) |
| 2 | **개선사항** (리오더 시 디테일 변경 → 디자인팀 확인 분기) |

→ 브랜드 발주서 UI에 **납기·개선사항** 필드 필수, 승인 R3 전 사전 입력

## 2.2 운영캘린더 · 기획전 · 팀 협업 (인터뷰 진행 중)

> **참고 UI:** BGROW OS 프로젝트 워크스페이스 (`os.bgrow.co.kr`) — 칸반·캘린더·타임라인·팀별 태스크  
> **Pain:** MD/마케팅/비주얼/물류/디자인 **팀별 캘린더 분산** → 소통 지연 · 누락 다수

### 해결 방향 (BGROW 참고 — **동일 복제 불필요**, 업무 상세 조회 중심)

```
[연간·반기 캘린더]  ← 한눈에: 기획전 일정 + 완료율(진행 정도)
        │
        ▼ 클릭
[기획전 = 프로젝트]  ← 채널·기간·할인·푸시상품·담당
        │
        ▼ 자동 전개 (R&R 템플릿)
[팀별 상세 업무]     ← MD / 마케팅 / 비주얼 / 물류 / 디자인 체크리스트
        │              칸반: **할 일 → 진행 중 → 완료** (기본 3단계, §2.8.1)
        ▼
[발주 연동]          ← 푸시 상품 → MD 선발주 · campaign-pl P&L
```

### BGROW 스크린샷에서 가져올 UI (이식 대상)

| BGROW 요소 | ATLM ERP 매핑 |
|------------|---------------|
| 프로젝트 워크스페이스 | **기획전 1건** = 프로젝트 |
| 탭: Task / AD / Contents / … | 팀·유형별 **업무 그룹** (R&R) |
| 칸반 (할일·진행·완료) | 팀별 태스크 보드 — 검토/F/U는 **태그** |
| 리스트 / **캘린더** / 타임라인 | 동일 3뷰 (`프로토타입_운영캘린더.html` + BGROW) |
| 업무 카드 (기간·담당·우선순위·태그) | `campaign_tasks` 레코드 |
| 필터 (상태·담당·우선순위) | 팀·담당자·브랜드 필터 |

### 캘린더 2층 구조 (핵심 요구)

| 레벨 | 보여주는 것 | 사용자 |
|------|------------|--------|
| **L1 캘린더** | 기획전 막대 · **전체 완료율** · D-day | 전사·경영 |
| **L2 프로젝트** | 팀별 업무 리스트 · 체크 · 칸반 | MD·마케팅·비주얼·물류·디자인 |

- 캘린더 자체 = **완료 정도** (빨갛게 안 끝난 팀 보임)
- 상세 = **기획전별 · 팀별** 체크리스트 (BGROW Task 탭)

### R&R 팀별 업무 템플릿 (기획서 §F·§P)

기획전 유형 선택 시 **자동 생성** (AI 보조 가능):

| 팀 | 예시 업무 |
|----|----------|
| **MD** | 푸시 SKU 확정 · 선발주 · 재고 확보 |
| **마케팅** | AD 소재 · 프로모션 세팅 · 채널 할인 등록 |
| **비주얼** | 룩북 · 상세페이지 · 배너 |
| **디자인** | 디테일 변경 확인 (리오더 시) |
| **물류** | 3PL 입고 확인 · 출고지시(이지어드민) |

### 팀별 상세 공유 UI — **템플릿 제안** (2026-07-10)

> **현황:** 팀별 고정 템플릿 없음. 스프레드시트·슬랙으로 각자 관리 → **ERP가 처음으로 표준화**

**형식:** **「항목 : 세부내용」** + 완료 체크 + (선택) 마감 D-day

**기획전 생성 시** 아래 항목이 **팀 탭에 자동 삽입** (빈 칸만 채우면 됨). 팀장이 항목 추가·삭제 가능.

**UI 동작:**
- 팀 탭 → 항목 행 (라벨 + 세부내용 + ✅ + D-day 뱃지)
- 미완료 + D-day 임박 → **해당 팀 Inbox** 알림
- 변경 시 **@멘션** (슬랙 대체)
- 첨부: 항목별 (배너 시안·랜딩 캡처 등)
- **완료율** = 팀별 체크 완료 / 전체 항목 → L1 캘린더에 표시

**슬랙·스프레드시트 대체 포인트:**
- 푸시 SKU **한 곳**에서 전 팀 조회 (Pain ⑦ 해결)
- 마감 D-day 자동 계산 (수기 캘린더 불필요)
- 기획전 종료 후 P&L 링크
- 기획전 생성 시 **푸시 품목을 전 팀에 공개** (MD만 아는 문제 해결)
- 발주 알림: 기획전 시작일 **-7주** → MD Inbox 「선발주 필요」
- 기획전 종료 후 → **P&L 탭** 자동 집계 (채널 매출·할인·광고·원가)

## 2.8 업무 OS — 협업·가시성·대시보드 (2026-07-10)

### 아키텍처 — 3층

```
[L1 캘린더]     연간·월간 — 기획전 일정 + 온보딩 상태 색상 + 완료율 링

      │
[위젯 대시보드]  직원별 조립 — 역할 기본값 + 개인 커스터마이즈
```

### 프로젝트(기획전) 상태 — **온보딩** 포함

| 상태 | 한글 | 의미 |
|------|------|------|
| `draft` | **미온보딩** | 아이디어만 · 캘린더 **미등록** · 팀 태스크 없음 |
| `onboarded` | **온보딩됨** | 캘린더에 올림 · 팀별 템플릿 **자동 전개** · 시작일 전 |
| `active` | **진행중** | 기간 내 · 팀 업무 진행 |
| `closed` | **마감** | 종료 · P&L 집계 · 읽기 전용 |

**온보딩 액션:** MD가 캘린더에 드래그 또는 「캘린더에 올리기」→ `draft` → `onboarded` + 팀 태스크 생성

캘린더 필터: `미온보딩` | `온보딩됨` | `진행중` | `마감` | `전체`

### 2.8.1 칸반 단계 — **추천안 (확정)**

> BGROW 5단계 **그대로 쓰지 않음**. 스프레드시트·슬랙 쓰던 팀에 **3단계가 맞음**.

#### 업무 유형별

| 유형 | UI | 상태 |
|------|-----|------|
| **팀 메모 항목** (`campaign_team_notes`) | 체크리스트 | ☐ / ✅ (2단계) |
| **담당 태스크** (`campaign_tasks`) | 칸반 | **할 일 · 진행 중 · 완료** |
| **R3 승인** | 별도 트랙 | 6단계 승인 엔진 (칸반과 분리) |

#### 칸반 3단계 + 태그

```
[ 할 일 ]  →  [ 진행 중 ]  →  [ 완료 ]
                  │
                  ├ 🏷 검토필요  (비주얼·MD 확인)
                  └ 🏷 F/U       (후속 조치)
```

- **검토 / F/U** = 칸반 **열 추가 X** → 「진행 중」카드에 **태그**
- 팀장 필터: `진행 중 + 검토필요`
- **5열 모드** (BGROW식) = Phase 3 팀 설정 **선택** (기본 OFF)

#### 확장 원칙 — **필요할 때 켜기**

> 기본은 **단순(3열·체크리스트)**. 팀·프로젝트별로 **옵션 추가** — 하드코딩 X.

| 레벨 | 예시 | 기본 |
|------|------|------|
| **팀 설정** | 5열 칸반 · 추가 태그 · 팀 전용 템플릿 항목 | OFF |
| **프로젝트** | 기획전마다 템플릿 항목 추가·삭제 | MD가 조정 |
| **시스템** | 슬랙 연동 · 광고비 외부연동 · 이지어드민 API | Phase별 ON |

```sql
team_work_settings (
  team,                      -- MD | MKT | VISUAL | ...
  kanban_mode,               -- 'simple_3' | 'extended_5'  default simple_3
  custom_tags jsonb,         -- 팀별 추가 태그
  template_overrides jsonb   -- 기획전 템플릿 항목 추가/삭제
)
```

- **1차 출시:** 3열 + 태그(검토/F/U)만
- **나중에:** 팀장이 설정에서 5열·커스텀 태그 켜기
- 새 기능 = **설정 플래그**로 추가 (전사 일괄 변경 없음)

### 4.4 홀세일 (B2 확정 — **해외만**)

> **국내 홀세일 없음.** 홀세일 = **해외 바이어** 납품만.

| 항목 | 규칙 |
|------|------|
| 범위 | **해외 홀세일 바이어**만 (국내 해당 없음) |
| 출고 | **이지어드민** (3PL 경유 없음) |
| 바이어 | `vendors` 지속 등록 (계속 증가) |
| ERP 역할 | 납품처 · 품목 · **인보이스** · **패킹리스트** · 정산 · 미수금 |

### §8.1 B3 — 왜 API 연동을 물었나

**목적:** 채널 판매 탭에서 **매일 판매·재고를 자동으로** 가져오기 위함.

| 연동 | ERP에서 쓰는 곳 |
|------|----------------|
| Cafe24 API | 자사몰 일간 매출·주문 |
| Shopify API | 해외몰 일간 매출·주문 |
| (W컨셉/29CM은 API 없음 → 엑셀+AI) | |

연동 가능하다면 Phase 2A에서 **수동 엑셀 없이** 자사몰·해외몰 판매 모니터링 가능.

### §8.2 B6 — 발주 분리 (확정)

**브랜드·OEM 발주서는 항상 분리.** LUMEN / AETALOOP / OEM — **3갈래 각각 별도 발주서.**

```
[LUMEN]     LUMEN MD → LUMEN 묶음 발주서 → 승인 → 생산발주
[AETALOOP]  AE MD    → AETALOOP 묶음 발주서 → 승인 → 생산발주
[OEM]       바이어   → 별도 양식 → 생산팀 생산발주
```

→ `qty_brand` / `qty_oem` 분리 필드 **불필요**. 기획서의 「OEM 합산」 케이스는 **미적용**.
